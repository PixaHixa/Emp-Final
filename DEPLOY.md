# نشر النظام (Industrial-Sys)

## 1) قاعدة البيانات Supabase (مرة واحدة)

1. افتح [Supabase](https://supabase.com) → مشروعك → **SQL Editor**
2. انسخ والصق محتوى الملف:
   ```
   supabase/migrations/00_deploy_all.sql
   ```
3. اضغط **Run**
4. تأكد في **Table Editor** من وجود:
   - `employees`
   - `attendance`
   - `manual_transfers`
   - `week_hour_adjustments`

إذا ظهرت رسالة حمراء في أعلى الموقع بعد النشر → الجداول ناقصة، أعد تشغيل الملف.

## 2) متغيرات البيئة (Vercel أو استضافة أخرى)

| المتغير | الوصف |
|---------|--------|
| `VITE_SUPABASE_URL` | رابط المشروع من Supabase → Settings → API |
| `VITE_SUPABASE_ANON_KEY` | المفتاح `anon` `public` |

**لا ترفع ملف `.env` على GitHub.** استخدم Environment Variables في لوحة الاستضافة.

محلياً: انسخ `.env.example` إلى `.env` واملأ القيم.

## 3) بناء ونشر الواجهة

```bash
npm install
npm run build
```

على **Vercel**: اربط المستودع، أضف المتغيرين أعلاه، Deploy.

## 4) موقع بصمة الحضور (الهاتف)

يجب أن يستخدم **نفس** مشروع Supabase. التفاصيل في:

`docs/PUNCH_INTEGRATION.md`

## 5) تحقق بعد النشر

- [ ] إضافة موظف من صفحة الموظفون
- [ ] بصمة حضور + مغادرة من الهاتف → يظهر الراتب في اللوحة
- [ ] ترحيل مبلغ + ساعة يدوية → تظهر بعد تحديث الصفحة على جهاز آخر
- [ ] لا يظهر شريط أحمر «قاعدة البيانات تحتاج إعداداً»

## 6) أمان (موصى به لاحقاً)

حالياً سياسات RLS مفتوحة (`allow all`) للتبسيط. للإنتاج طويل الأمد: قيّد الصلاحيات حسب دور المستخدم.
